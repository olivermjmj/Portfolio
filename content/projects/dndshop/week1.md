# Week 1 – Project Ideation and Planning

## Sessions
- Brainstormed three possible portfolio project ideas.
- Analyzed feasibility and alignment with semester requirements.

## Goals
- Generate 2–3 viable project ideas.
- Evaluate technical complexity and academic relevance.

## Decisions Made
- Dropped the idle game idea due to limited web relevance.
- Dropped the booking system due to insufficient complexity.
- Selected the D&D Shop Backend as the final project concept.

## Work Completed
- Wrote short descriptions of each idea.
- Documented pros and cons.

## Technical Focus
- Project scope evaluation.
- System complexity assessment.

## Challenges
- Balancing personal interest with academic requirements.
- Ensuring sufficient architectural depth for a semester-long project.

## Solutions
- Chose a concept with strong domain complexity and scalability.

## Reflection
The structured evaluation clarified which idea met both motivational and academic requirements.  
The D&D Shop provides sufficient architectural depth for a production-ready backend project.

## Next Steps
- Define core domain entities.
- Design an initial ERD.
